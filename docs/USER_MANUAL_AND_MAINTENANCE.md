# 校园二手集市用户手册与维护说明

## 1. 用户手册

### 1.1 系统入口

本系统为校园二手物品交易 Web 平台，前端默认运行在：

```text
http://localhost:5173
```

后端默认运行在：

```text
http://localhost:8080
```

开发演示环境使用一卡通号和邮箱验证码登录，验证码开发期固定为：

```text
123456
```

### 1.2 登录与身份认证

1. 打开前端页面，进入登录页。
2. 输入 9 位一卡通号。
3. 点击“获取验证码”。
4. 输入验证码 `123456`。
5. 点击“验证并登录”。
6. 首次登录用户会进入新人引导页，老用户进入首页。

注意事项：

- 一卡通号必须为 9 位数字。
- 如果验证码错误或过期，系统会提示登录失败。
- 演示环境使用固定验证码，正式环境应接入真实校园邮箱或统一身份认证。

### 1.3 新人引导与资料完善

首次登录后需要完善资料：

1. 输入昵称。
2. 选择校区、苑区、楼栋。
3. 可填写微信号，便于线下面交沟通。
4. 保存后进入集市首页。

资料完善后，用户才可以发布商品。

### 1.4 浏览和搜索商品

首页提供商品列表和搜索功能：

1. 顶部搜索框输入关键词。
2. 可按分类筛选，例如专业书籍、电子数码、宿舍日用。
3. 点击商品卡片进入详情页。

商品卡片会展示图片、标题、价格、成色、想要人数和卖家信息。

### 1.5 查看商品详情

商品详情页包括：

- 商品图片轮播；
- 商品标题、价格、成色；
- 详细描述；
- 浏览人数和想要人数；
- 卖家信息和面交地点；
- 收藏、私信留言、我想要按钮。

如果商品已经被预定或已完成，底部预定按钮会被禁用或展示对应状态。

### 1.6 发布商品

卖家发布商品步骤：

1. 点击底部“发布”。
2. 上传 1 到 6 张商品图片。
3. 填写商品标题。
4. 选择商品分类和成色。
5. 输入出售价格。
6. 选择面交地点。
7. 填写详细描述。
8. 点击“立即发布”。

发布规则：

- 至少上传一张图片；
- 标题最多 40 个字符；
- 价格必须大于 0；
- 描述最多 500 个字符；
- 面交地点必须选择到楼栋。

### 1.7 收藏商品

在商品详情页点击“收藏”即可收藏商品。收藏后的商品可在个人中心“我的收藏”中查看。

限制：

- 用户不能收藏自己发布的商品；
- 已下架商品不能被收藏。

### 1.8 私信沟通

买家可以在详情页点击“私信留言”进入聊天页面。

聊天支持：

- 发送文字消息；
- 引用消息回复；
- 2 分钟内撤回自己发送的消息；
- 查看未读消息；
- 删除自己的会话入口。

限制：

- 卖家不能和自己发起私信；
- 非会话参与者不能查看或发送消息；
- 不能引用其他会话中的消息；
- 不能引用已撤回消息。

### 1.9 预定商品

买家在商品详情页点击“我想要”后，系统会弹出二次确认。

确认后：

1. 商品状态从“在售”变为“被预定”。
2. 系统创建订单。
3. 系统创建或复用买卖双方的私信会话。
4. 卖家收到商品被预定通知。
5. 买家自动进入聊天页沟通面交。

限制：

- 卖家不能预定自己发布的商品；
- 已被预定、已完成、已下架商品不能再次预定；
- 多个买家同时预定时只能成功一个。

### 1.10 取消预定和完成交易

当前后端支持：

- 买家或卖家取消未完成预定；
- 卖家确认交易完成；
- 买家不能确认完成；
- 非订单参与者不能操作订单。

交易完成后：

- 订单状态变为 `COMPLETED`；
- 商品状态变为“已完成”；
- 买家可以在个人中心“我买到的”中查看；
- 买家收到交易完成通知。

### 1.11 个人中心

个人中心包括：

- 用户头像、昵称、常驻地点、一卡通号；
- 我发布的；
- 我买到的；
- 我的预定；
- 我的收藏；
- 编辑资料；
- 退出登录。

用户可以在个人中心更新昵称和微信号。

## 2. 管理与维护说明

### 2.1 本地启动

后端启动：

```bash
cd backend
./mvnw spring-boot:run
```

前端启动：

```bash
cd frontend
npm run dev
```

### 2.2 测试与回归

每次修改后建议执行：

```bash
cd backend
./mvnw test
```

```bash
cd frontend
npm run build
```

性能压测：

```bash
node scripts/qa-load-test.js
```

### 2.3 数据库维护

开发和测试环境默认使用 H2 内存数据库，启动时由 `DataInitializer` 初始化地点树和默认用户。

如需 MySQL 部署：

1. 使用 `application-mysql.yml` 配置数据库连接。
2. 确认 `app.security.dev-auth-enabled=false`。
3. 根据 `backend/src/main/resources/db/schema.sql` 检查表结构。
4. 重新执行后端自动化测试和并发压测。

### 2.4 安全维护

| 项目 | 维护要求 |
|---|---|
| JWT 密钥 | 正式环境必须替换默认密钥 |
| 开发 Header | 正式环境必须关闭 `dev-auth-enabled` |
| 上传目录 | 限制图片大小、格式和访问路径 |
| 用户权限 | 商品、订单、会话都必须校验当前用户身份 |
| 错误响应 | 不向前端暴露异常堆栈 |

### 2.5 性能维护

| 场景 | 建议 |
|---|---|
| 商品列表变慢 | 增加分页、按状态和创建时间建索引 |
| 搜索变慢 | 给标题、分类、状态字段增加索引或接入搜索服务 |
| 图片加载慢 | 增加缩略图和图片压缩 |
| 预定冲突多 | 保留数据库事务和锁，MySQL 部署后压测验证 |
| 消息请求频繁 | 增加分页、轮询间隔或 WebSocket |

### 2.6 后续迭代建议

| 功能 | 说明 |
|---|---|
| 举报审核 | 新增 Report 实体、举报接口、管理员处理接口和处理日志 |
| 管理员后台 | 支持查看举报、下架违规商品、封禁异常用户 |
| 预定 TTL | 被预定后超过约定时间未确认可自动恢复在售 |
| E2E 测试 | 使用 Playwright 覆盖登录、发布、搜索、预定、聊天 |
| 部署监控 | 增加接口日志、错误率、响应时间和数据库连接监控 |

### 2.7 常见问题

| 问题 | 处理方式 |
|---|---|
| 登录提示验证码错误 | 确认已点击获取验证码，开发期使用 `123456` |
| 发布商品失败 | 检查是否完善资料、是否上传图片、价格是否合法 |
| 看不到商品 | 确认商品是否已下架、已完成或搜索条件过窄 |
| 无法预定 | 商品可能已被别人预定或已完成 |
| 消息无法发送 | 检查是否为会话参与者、内容是否为空或超长 |
| 压测失败 | 确认后端已启动，且开发期 Header 未关闭 |
