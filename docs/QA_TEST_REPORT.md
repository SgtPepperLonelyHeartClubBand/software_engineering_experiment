# 校园二手集市测试报告

## 1. 报告概述

| 项目 | 内容 |
|---|---|
| 项目名称 | 校园二手集市 Web 平台 |
| 负责角色 | 测试与 QA |
| 报告日期 | 2026-06-20 |
| 测试对象 | Spring Boot 后端、Vue/Vite 前端、核心交易业务 |
| 测试目标 | 验证功能、状态机、权限、异常场景、并发预定和基础性能 |

本轮测试以“校园内二手交易闭环”为核心，从买家、卖家和系统安全角度验证主要功能。测试重点不是只检查页面能否打开，而是验证后端业务规则是否能防止一物多卖、越权操作和异常状态流转。

## 4. 新增 QA 测试说明

### 4.1 异常交易场景

新增 `QaExceptionScenarioTests` 后，系统额外验证以下场景：

| 场景 | 期望 |
|---|---|
| 卖家预定自己的商品 | 返回 `code=400` |
| 路人取消别人的订单 | 返回 `code=403` |
| 买家确认完成订单 | 返回 `code=403` |
| 已完成订单再次取消 | 返回 `code=400` |
| 已完成商品再次预定 | 返回 `code=400` |
| 已下架商品被预定/收藏/私信 | 返回 `code=404` |

### 4.2 非法输入场景

| 场景 | 期望 |
|---|---|
| 发布商品缺少图片 | 返回 `code=400` |
| 发布商品价格小于 0.01 | 返回 `code=400` |
| 发布商品标题超过 40 字 | 返回 `code=400` |
| 发送空消息 | 返回 `code=400` |
| 发送超过 200 字消息 | 返回 `code=400` |

### 4.3 私信安全场景

| 场景 | 期望 |
|---|---|
| 非会话参与者查看消息 | 返回 `code=403` |
| 非会话参与者发送消息 | 返回 `code=403` |
| 跨会话引用消息 | 返回 `code=400` |
| 撤回他人消息 | 返回 `code=403` |
| 引用已撤回消息 | 返回 `code=400` |

### 4.4 并发预定场景

新增 `QaConcurrencyTests`，使用 6 个买家线程同时请求同一商品的预定接口。测试期望为：

- 成功数量为 1；
- 业务拒绝数量为 5；
- 商品最终状态为“被预定”；
- 商品 `wantCount=1`；
- 存在一个 `RESERVED` 状态订单。

该测试对应项目中“一物多卖”风险，是本轮 QA 的核心测试点。

## 7. 缺陷与风险

| 编号 | 类型 | 描述 | 处理建议 |
|---|---|---|---|
| RISK-001 | 后续扩展 | 举报审核和管理员处理暂未纳入当前交付闭环 | 后续新增 Report/Admin 模块后补测 |

## 8. 验收结论

在当前实现范围内，系统已经具备校园二手交易的核心闭环：用户登录后可以发布、搜索、查看、收藏、私信、预定和完成交易；后端通过统一鉴权、业务校验、事务和悲观锁防止主要异常交易风险。

当前可交付版本可以满足课程演示和基础验收要求。举报审核、管理员治理和预定超时释放属于后续增强项，已在本报告中作为迭代测试项记录。
